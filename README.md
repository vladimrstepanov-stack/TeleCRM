# Voice CRM MVP

Асинхронный Telegram-бот для риелтора. Любое голосовое или текстовое сообщение
интерпретируется языковой моделью как **пополнение данных** или **запрос данных**,
сохраняется в MySQL с историей изменений и превращается в редактируемые
DOCX-документы.

- Голос → текст: `qwen3-asr-flash` через AiTunnel (OpenAI-совместимый API).
- Интерпретация и извлечение сущностей: DeepSeek через AiTunnel.
- Доступ только у Telegram ID из `ALLOWED_TELEGRAM_IDS` (остальным — короткий отказ
  без вызова платного AI).

## Требования

- Python 3.11+
- MySQL 8+ (локально для разработки; сервер Beget — позже)

## Установка (Windows PowerShell)

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev]"
```

На Linux/macOS активация: `source .venv/bin/activate`.

## Переменные окружения

Скопируйте `.env.example` в `.env` и заполните значения. `.env` **никогда** не
коммитится (см. `.gitignore`).

| Переменная | Назначение |
|---|---|
| `ALLOWED_TELEGRAM_IDS` | Разрешённые Telegram ID через запятую |
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather |
| `DATABASE_URL` | Строка подключения MySQL (async, драйвер aiomysql) |
| `AITUNNEL_BASE_URL` | Базовый URL AiTunnel (OpenAI-совместимый) |
| `AITUNNEL_API_KEY` | Ключ AiTunnel |
| `STT_MODEL` | Модель распознавания речи (`qwen3-asr-flash-...`) |
| `DEEPSEEK_MODEL` | Модель интерпретации (`deepseek-chat`) |
| `LOG_LEVEL` | Уровень логов (INFO по умолчанию) |

### Про адрес MySQL `10.16.0.2`

Это **внутренний** адрес в сети Beget. С домашнего компьютера он, скорее всего,
**недоступен**. Поэтому:

- для локальной разработки используйте свою локальную MySQL в `DATABASE_URL`
  (например, `mysql+aiomysql://telecrm:pass@127.0.0.1:3306/telecrm?charset=utf8mb4`);
- на сервере будет отдельный `.env` с теми же именами переменных, но серверными
  значениями (внутренний адрес там доступен).

Если локальной MySQL нет, схему и тесты можно проверять на SQLite (тесты уже так и
работают), а бот запускать после появления доступной MySQL.

## Создание схемы БД

Через Alembic (рекомендуется):

```powershell
alembic upgrade head
```

Или быстрым скриптом для разработки:

```powershell
python -m scripts.init_db
```

## Запуск бота

```powershell
python -m app.main
```

Бот работает в режиме long polling.

## Логи

Пишутся с первого запуска в папку `logs/` (в Git не попадает):

- `logs/app.log` — общий журнал (вход, intent, этапы STT → LLM → БД → DOCX);
- `logs/error.log` — только ошибки.

Просмотр в реальном времени:

```bash
tail -f logs/app.log
tail -f logs/error.log
```

Секреты (токены, ключи, пароли) в логи не записываются.

## Сценарий использования

1. Пришлите голосом/текстом новые данные: «Новый клиент Сидоров Василий,
   трёхкомнатная квартира, 5 этаж, продаётся за 5 млн» → бот сохранит карточку
   (клиент + объект + сделка) и покажет три кнопки.
2. Запросите данные: «Отчёт по квартире Сидорова» → бот найдёт карточку.
   - Если не найдено — попросит телефон/идентификаторы.
   - Если найдено — покажет карточку и кнопки **Обновить / К встрече / Отчёт**.
3. **Обновить** — следующее сообщение применяется к текущей карточке (цена, статус,
   показ + обратная связь, оферта).
4. **Отчёт** — DOCX «Отчёт собственнику» за весь срок (актуальное сверху, хронология
   вниз). **К встрече** — DOCX-сводка для передачи клиенту.
